# 选项跳转
## 功能概述
该功能需与**分支功能**搭配使用，核心作用是在交互选项标签中配置跳转逻辑，点击对应选项后可直接跳转到指定的分支节点。

## 语法规范
语法格式根据框架版本区分，需严格匹配自身使用的版本号进行编写：
### 2.2及以上版本
```
choice:
    "选项内容" 分支名称
```
### 2.2以下版本
```
choice "选项内容" [分支名称]
```

## 参数说明
| 参数       | 是否必需 | 示例         | 说明                     |
|------------|----------|--------------|--------------------------|
| 选项内容   | 是       | 选择咖啡     | 交互时展示的选项文本内容 |
| 分支名称   | 是       | coffee_choice | 需跳转的目标分支唯一标识 |

## 代码示例
### 2.2及以上版本示例
```
choice:
    "Coffee1111111111111111" coffee_choice
    "Tea11111111111111111111" tea_choice
```

## 注意事项
1. 分支名称需与项目中定义的分支标识完全一致，区分大小写，且不可包含空格、特殊符号；
2. 选项内容需用英文双引号包裹，支持常规的文本字符；
3. 2.2及以上版本中，`choice:`后需换行，每个选项单独一行并保持缩进规范，保证语法解析正常。
